CXP Protocol

CXP is an example of point-to-point CCR transport between EHR systems. MedCommons has placed CXP into the public domain in order to stimulate comment from EHR vendors and other interested parties.

The CXP protocols are demonstrated by a Firefox extension (PC/Mac) that makes it convenient to cross firewalls separating EHR applications by using a Web accessible CXP server as intermediary. You are free to install your own CXP server from the public domain distribution.

MedCommons operates an enhanced CXP server (https://gateway001.private.medcommons.net:8443/router/CxpServlet) that anyone can use. This server will send a MedCommons email notification based on any emails found in the TO, FROM or PATIENT actors of a CCR. MedCommons tracks this email activity. We hope you will use the MedCommons server as much as possible - it's free - please send feedback to cmo@medcommons.net or to the CCR Accelerator Group.

This note, and the public domain CXP software it refers to is located at https://secure.medcommons.net/mc/cxp

The documentation for the CXP protocol is at https://secure.medcommons.net/mc/cxp/collateral/cxprotocol.pdf

CCR Client

The CCR Client demonstrates EHR to EHR transport using the CXP Protocol. A CCR saved from a EHR is picked up by the local CCR Client, transported by MedCommons using CXP and becomes available to any other EHR that has a CCR Client installed.

MedCommons, acting as the intermediary transport agent, facilitates interoperability testing by adding:

  • an email notification feature based on Actors in the CCR itself,
  • an online CCR viewer, and
  • a central problem blog to facilitate community-based resolution of incompatibilities and other interoperability problems.

The CCR Client is just an interoperability testing tool. It is assumed that all CCRs are completely anonymous and created explicitly to demonstrate the implementation of various features of the ASTM CCR standard by participating EHR vendors.

CCR Client Installation

  1. Install Firefox and the CCR Client Firefox Extension
    • Download and install Firefox (FF go to www.mozilla.org to get a copy). Please ensure yor Firefox is at version 1.0.6 or later.
    • Download https://secure.medcommons.net/mc/cxp/mccxp_0.9.5.xpi to your hard disk.
    • Open the extension file with Firefox and click the Install button when it appears
    • Note: you must have "allow websites to install plug-ins" checked in your prefs
    • After installation, be sure to restart Firefox completely - all windows must close for the extension to install properly.
    • When installation is successful and the client is installed, an option labelled "CCR Client" will appear in the Tools menu as pictured below:
  2. Prepare a CCR with Embedded Email Address(es) (optional)
    • If you are utilizing the MedCommons test server as preset in the CCR Client Settings, a notification email can be sent to the TO, FROM or PATIENT Actors in the CCR when your CCR is processed. If you are using the barebones CXP public domain software, the contents of the CCR will be ignored.
    • Create or open a CCR in your EHR. CCR samples are in https://secure.medcommons.net/mc/cxp/collateral. /li>
    • If possible, enter a notification email address for MedCommons to use, otherwise you will have an opportunity to enter a notification address in Step 3.
    • Save the file as myccr.xml (or another name if you like) on your local file system
  3. Run CCR Client to Send CCR
    • Open FF if not already open, go to Tools>CCR Client
    • Accept the PKI Certificates if offered.
    • Browse to the file myccr.xml wherever it is stored in local file system
    • Enter the email address of the party you would like MedCommons to notify, and click Send. The CCR Client will add or change the email address of the TO Actor accordingly.
    • When accepted, note that a Transaction ID and the PIN are added to your History list
  4. View the CCR via MedCommons
    • Double click the entry in History to preview what others will see the CCR online.
    • Anyone can view the CCR you sent by clicking on the Tracking Number link in the Notification email sent by MedCommons, by clicking in the community blog at https://secure.medcommons.net/ui/cxpblog.php or by entering the Tracking Number as displayed in History on the MedCommons Home Page at https://secure.private.medcommons.net.
    • They will need to get the PIN from you by some means and enter it when challenged to prove that they are your intended recipient.
  5. Get the CCR into Destination EHR
    • The destination needs to install Firefox and the CCR Client as in step 1, above.
    • Click the Get. button
    • Enter Tracking ID and PIN
    • Enter a somename.xml and click Save
    • Open somename.xml in the EHR
  6. Report Problems
    • The "Report a Problem with a CCR" link on the CCR Client will add the Tracking ID and PIN along with your free text comment to a running blog that is accessible to all.
    • Do not post a CCR or a report that you don't want to share with the world.
    • You may review the blog at https://secure.private.medcommons.net/ui/cxpblog.php

Please send feedback and comments to cmo@medcommons.net
