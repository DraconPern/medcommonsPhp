MedCommons Support for Trackers

This note describes the implementation of Trackers within MedCommons. Trackers are implemented as a MedCommons Extension. Code for Trackers is in /trunk/php/trackers

What is a Tracker?

At this point, a Tracker is just a named object that maintains a timeseries history of values. It is associated with a MedCommons Account. Each user, within his own account, accumulates an ever increasing collection of objects, representing weight, blood pressure, glucose and blood test results, and more. A user adds each named Tracker to his account at his discretion, normally utilizing the Tracker Editor and selecting from a dropdown list

Third parties can create libraries of additional trackers, either by simply building forms (either handmade or via xforms) or by writing their own code (e.g. for data acquisition or display) MedCommons can sell, rent, and bill for usages of these libraries. These third party libraries are also implemented as MedCommons Extensions

Adding Tracker Support to Your MedCommons Account

Just get to the Extensions Page via:
My Account > Apps and Services > Add Tracker Support

You must add tracker support before you add any other Tracker Extension libraries from the Apps and Services Page

Using Trackers in Your Account

You must be logged on to your account to use Trackers, unless you have hardwired the Tracker Gadget to your account. You can view your Trackers anytime.

Use the editor to choose and personalize your Trackers. You can set many properties of both the display system and the means of data acquisiton.

You can also utilize Trackers via your personal Google Homepage.

Defining New Trackers

There are two steps to making new trackers available on MedCommons.
  • Define a Catalog of your New Trackers
  • Wire the Catalog into the MedCommons Apps and Services Extension system

The catalog is simply a list of trackers, their units, and some other simple properties. Here are some catalogs: blood catalog, gps catalog, and finally the builtins that get autoloaded when the Tracker Extension is installed., Note that these are just plain URLs, and might point to any website, not necessarily inside MedCommons

Wiring in Extensions to MedCommons is described here but we provide links to the blood extension and gps extension. There is no extension for the builtins. You need to make sure the 'id' field' in both the Catalog xml file and the Extension xml file are identical

Add Your Own Code to the Tracker System

You can extend the Tracker System in various ways:
  • You can plug in your own xForms compatible remote input source to deliver Tracker Messages into your account
  • You can write your own Graphics code that runs inside the user's Account
  • You can write a custom viewer in PHP or Java that completely replaces the builtin tracker display
  • You can write a background program that transparently updates the users Trackers as needed
It's a little vague right now on how to do this on a personal level, but all of the callbacks are in place in both the catalogs and extension xml files. You can just edit these xml files to play with this for now.

Adding 3rd Party Code to the Trackers

Most users will not add their own code, but rather buy or rent code from 3rd parties. MedCommons acts as a broker in this situation, taking a service fee for providing access to the account and for billing.
